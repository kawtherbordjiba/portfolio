Project 10

Sorting Algorithm Program

C Algorithms Complexity Analysis Divide & Conquer Recursion

Overview

A comprehensive C program implementing and comparing five classic sorting algorithms: Bubble Sort, Selection Sort, Insertion Sort, Merge Sort, and Quick Sort. The program accepts an array from the user, runs each algorithm, and displays the sorted output alongside a complexity analysis. This project was designed to build a deep understanding of algorithmic efficiency, recursion, and the practical differences between O(n²) and O(n log n) approaches.

Algorithms Implemented

Bubble Sort
O(n²) avg — O(n) best
Repeatedly swaps adjacent elements that are out of order. Simple but inefficient. Optimized with an early-exit flag when no swaps occur in a pass.
Selection Sort
O(n²) always
Finds the minimum element in the unsorted region and places it at the front. Consistent performance regardless of input order. Minimal swaps.
Insertion Sort
O(n²) avg — O(n) best
Builds a sorted sub-array one element at a time. Efficient for nearly sorted data — used by QuickSort as a base case for small partitions.
Merge Sort
O(n log n) always
Divide and conquer — recursively splits the array, then merges sorted halves. Stable sort with guaranteed performance. Requires O(n) extra space.
Quick Sort
O(n log n) avg — O(n²) worst
Partition-based divide and conquer. Selects a pivot and arranges elements into two sub-arrays. Fastest in practice for most inputs. In-place sort.

Core Implementations

/* Bubble Sort — O(n²) avg, O(n) best with early exit */
void bubbleSort(int arr[], int n) {
    int i, j, temp, swapped;
    for (i = 0; i < n - 1; i++) {
        swapped = 0;
        for (j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                /* Swap adjacent elements */
                temp        = arr[j];
                arr[j]      = arr[j + 1];
                arr[j + 1]  = temp;
                swapped     = 1;
            }
        }
        /* If no swap occurred, array is already sorted */
        if (!swapped) break;
    }
}
/* Insertion Sort — O(n²) avg, O(n) best (nearly sorted input) */
void insertionSort(int arr[], int n) {
    int i, key, j;
    for (i = 1; i < n; i++) {
        key = arr[i];
        j   = i - 1;
        /* Shift elements greater than key one position right */
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;  /* Insert key in correct position */
    }
}
/* Selection Sort — O(n²) always, minimal swaps */
void selectionSort(int arr[], int n) {
    int i, j, minIdx, temp;
    for (i = 0; i < n - 1; i++) {
        minIdx = i;
        /* Find the minimum element in unsorted region */
        for (j = i + 1; j < n; j++)
            if (arr[j] < arr[minIdx]) minIdx = j;

        /* Swap minimum with first element of unsorted region */
        if (minIdx != i) {
            temp         = arr[minIdx];
            arr[minIdx]  = arr[i];
            arr[i]       = temp;
        }
    }
}
/* Merge Sort — O(n log n) guaranteed, divide and conquer */
void merge(int arr[], int l, int m, int r) {
    int n1 = m - l + 1, n2 = r - m;
    int L[n1], R[n2], i = 0, j = 0, k = l;

    for (int a = 0; a < n1; a++) L[a] = arr[l + a];
    for (int b = 0; b < n2; b++) R[b] = arr[m + 1 + b];

    /* Merge the two sorted halves */
    while (i < n1 && j < n2)
        arr[k++] = (L[i] <= R[j]) ? L[i++] : R[j++];
    while (i < n1) arr[k++] = L[i++];
    while (j < n2) arr[k++] = R[j++];
}

void mergeSort(int arr[], int l, int r) {
    if (l < r) {
        int m = l + (r - l) / 2;  /* Avoid overflow */
        mergeSort(arr, l, m);
        mergeSort(arr, m + 1, r);
        merge(arr, l, m, r);
    }
}
/* Quick Sort — O(n log n) avg, O(n²) worst (sorted input) */
int partition(int arr[], int low, int high) {
    int pivot = arr[high];  /* rightmost element as pivot */
    int i = low - 1, temp;

    for (int j = low; j < high; j++) {
        if (arr[j] < pivot) {
            i++;
            temp    = arr[i]; arr[i] = arr[j]; arr[j] = temp;
        }
    }
    /* Place pivot in its correct position */
    temp        = arr[i + 1];
    arr[i + 1]  = arr[high];
    arr[high]   = temp;
    return i + 1;
}

void quickSort(int arr[], int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);   /* Sort left of pivot */
        quickSort(arr, pi + 1, high);  /* Sort right of pivot */
    }
}

What I Learned

This project gave me a practical and theoretical grasp of algorithmic complexity. Seeing all five algorithms run on the same input made the difference between O(n²) and O(n log n) concrete — not just on paper. I also deepened my understanding of recursion through Merge Sort and Quick Sort, and learned how to analyze best, average, and worst cases separately. The stability property of sorting algorithms and its implications for real-world use was another key takeaway.
