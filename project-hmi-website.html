Project 06

ErgoForm — HMI Ergonomic Form Evaluator

HTML CSS JavaScript HMI ISO 9241 WCAG Ergonomics

Overview

A browser-based tool that evaluates the ergonomic quality of a form design against established HMI principles — including ISO 9241 usability standards, Shneiderman's 8 Golden Rules, and WCAG accessibility guidelines. The user configures a form (colors, number of fields, labels, validation, feedback) and the tool instantly computes a score out of 100 with a detailed per-criterion breakdown and a live visual preview of the form. The goal: turn HMI theory into something measurable and actionable.

Key Features

  • Real WCAG contrast ratio computed from hex colors using the official sRGB luminance formula
  • 13 ergonomic criteria evaluated — from label clarity to responsive layout
  • Animated score ring counting up to the final result, color-coded green / yellow / red
  • Per-criterion feedback explaining what passed, what failed, and exactly why it matters
  • Live form preview rendered with the user's chosen colors, font size, and field count
  • Toggle checklist to simulate different form configurations instantly
  • Shneiderman's Rules #5 (error prevention) and #6 (informative feedback) directly applied
  • Runs entirely client-side — no backend, no dependencies, single HTML file

The 13 Ergonomic Criteria

01Color Contrast (WCAG)
Contrast ratio computed from background and text colors. Rated AAA (≥7:1), AA (≥4.5:1), or fail.
02Clear Field Labels
Every field must have a descriptive label — core ISO 9241 requirement. Missing labels score zero.
03Input Guidance
Placeholder text guiding expected format — e.g. DD/MM/YYYY, +213 XX XX XX XX.
04Required Field Markers
Mandatory fields must be visually marked to prevent confusing post-submit errors.
05Input Validation
Real-time or on-submit validation prevents incorrect data from reaching the backend.
06Error Message Clarity
Plain-language error messages. "Password must be 8 characters" beats a silent red border.
07Logical Field Grouping
Related fields grouped together reduce cognitive load and improve scan time.
08Form Length
Under 5 fields is ideal. Over 12 triggers an abandonment risk warning.
09Font Size
16px+ is ideal for all users. Below 12px is flagged as inaccessible.
10Form Width
Medium-width forms optimize reading line length. Too wide or too narrow both lose points.
11Submit Confirmation
Feedback after submission — applying Shneiderman's Rule #6 (informative feedback).
12Responsive Layout
Form must adapt to mobile screens — over 60% of web traffic is on smartphones.
13Progress Indicator
For forms with 6+ fields, a step indicator is required to prevent user abandonment.

HMI Theory Applied

  • ISO 9241 — Usability standards for interactive systems: effectiveness, efficiency, satisfaction
  • Shneiderman's 8 Golden Rules — Feedback, error prevention, consistency, user control
  • WCAG 2.1 — Contrast ratio thresholds: AA (4.5:1) and AAA (7:1) for normal text
  • Nielsen's 10 Heuristics — Visibility of system status, user control, error recovery
  • Cognitive Load Theory — Grouping fields and minimizing simultaneous decisions
  • Fitts' Law — Button size and positioning for efficient, low-error interaction

What I Learned

Building a tool that evaluates HMI quality forced me to understand the theory at a much deeper level than simply applying it. Implementing the WCAG contrast ratio formula from scratch — converting hex colors to relative luminance values using the official sRGB formula — was particularly rewarding. This project made me realize that HMI ergonomics is not about aesthetics alone: it is a measurable, engineering discipline with real standards and formulas behind every design decision.
