Project 03

Java CORBA

Java C CORBA IDL ORB Middleware Distributed Systems

Overview

Implementation of a distributed vowel-counting service using CORBA (Common Object Request Broker Architecture). Unlike RMI which is Java-only, CORBA uses a language-neutral Interface Definition Language (IDL) to define the service contract, enabling a Java server to be consumed by both a Java client and a C client — demonstrating true cross-language distributed computing through an Object Request Broker (ORB).

CORBA vs RMI — Key Difference

Java RMI

  • Java only
  • Interface defined in Java
  • RMI Registry for lookup
  • Simpler setup

CORBA

  • Language independent
  • Interface defined in IDL
  • CosNaming Service for lookup
  • Java, C, C++, Python clients

How It Works — Development Flow

1
Write the IDL file IDL
Define the service interface in language-neutral IDL — VoyelleService.idl
2
Compile IDL → generate stubs & skeletons
idlj -fall VoyelleService.idl → generates Java stub/skeleton. idl_to_wsdl or TAO for C.
3
Implement the server Java
Extend the generated VoyelleServicePOA skeleton and implement the business logic.
4
Write the client Java C
Both clients initialize an ORB, resolve the naming service, and call the remote method using their language's generated stub.
5
Run with ORB daemon
Start orbd (or tnameserv), then launch server, then any client — in any language.

Key Features

  • IDL-defined interface — the single source of truth for all languages
  • Java server using POA (Portable Object Adapter) model
  • CosNaming service for object lookup and registration
  • Java client consuming the remote service via generated stub
  • C client demonstrating true cross-language interoperability via IIOP
  • ORB handles marshaling, transport, and object lifecycle transparently

// VoyelleService.idl
// Language-neutral contract — compiled to Java stubs AND C stubs

module VoyelleApp {

    interface VoyelleService {

        /**
         * Counts vowels (a, e, i, o, u, y) in the given string.
         * @param texte  The input string
         * @return       Number of vowels found
         */
        long compterVoyelles(in string texte);

    };

};
import VoyelleApp.VoyelleServicePOA;
import org.omg.CORBA.ORBPackage.InvalidName;

/**
 * Server-side implementation — extends the generated POA skeleton.
 * The ORB routes incoming calls from ANY language client to this class.
 */
public class VoyelleImpl extends VoyelleServicePOA {

    @Override
    public int compterVoyelles(String texte) {
        if (texte == null || texte.isEmpty()) return 0;

        int compteur = 0;
        String min = texte.toLowerCase();

        for (int i = 0; i < min.length(); i++) {
            char c = min.charAt(i);
            if (c == 'a' || c == 'e' || c == 'i' ||
                c == 'o' || c == 'u' || c == 'y') {
                compteur++;
            }
        }

        System.out.println("[CORBA Server] \"" + texte + "\" → " + compteur);
        return compteur;
    }
}
import VoyelleApp.VoyelleService;
import org.omg.CORBA.*;
import org.omg.CosNaming.*;
import org.omg.CosNaming.NamingContextPackage.*;
import org.omg.PortableServer.*;

/**
 * Starts the ORB, activates the servant via POA,
 * and registers it in the CosNaming service as "VoyelleService"
 */
public class VoyelleServer {

    public static void main(String[] args) {
        try {
            // 1 — Initialize the ORB
            ORB orb = ORB.init(args, null);

            // 2 — Get the root POA and activate it
            POA rootPOA = POAHelper.narrow(
                orb.resolve_initial_references("RootPOA"));
            rootPOA.the_POAManager().activate();

            // 3 — Create and activate the servant
            VoyelleImpl servant = new VoyelleImpl();
            rootPOA.activate_object(servant);

            // 4 — Register in the Naming Service
            org.omg.CORBA.Object objRef =
                orb.resolve_initial_references("NameService");
            NamingContextExt ncRef = NamingContextExtHelper.narrow(objRef);

            NameComponent[] path = ncRef.to_name("VoyelleService");
            ncRef.rebind(path, rootPOA.servant_to_reference(servant));

            System.out.println("✓ CORBA Server ready — VoyelleService registered");
            System.out.println("  Accepting Java and C clients...");

            // 5 — Block and serve requests
            orb.run();

        } catch (Exception e) { e.printStackTrace(); }
    }
}
import VoyelleApp.*;
import org.omg.CORBA.*;
import org.omg.CosNaming.*;
import java.util.Scanner;

/**
 * Java CORBA client — resolves VoyelleService from the naming
 * service and calls compterVoyelles() remotely via ORB/IIOP
 */
public class VoyelleClient {

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        try {
            // 1 — Initialize the ORB
            ORB orb = ORB.init(args, null);

            // 2 — Locate the Naming Service
            org.omg.CORBA.Object objRef =
                orb.resolve_initial_references("NameService");
            NamingContextExt ncRef = NamingContextExtHelper.narrow(objRef);

            // 3 — Resolve the service and narrow to the correct type
            VoyelleService service =
                VoyelleServiceHelper.narrow(ncRef.resolve_str("VoyelleService"));
            System.out.println("✓ Service CORBA trouvé\n");

            // 4 — Remote method invocation loop
            boolean continuer = true;
            while (continuer) {
                System.out.print("Entrez une chaîne : ");
                String chaine = scanner.nextLine();

                int resultat = service.compterVoyelles(chaine);
                System.out.println("→ Voyelles : " + resultat + "\n");

                System.out.print("Continuer ? (o/n) : ");
                String rep = scanner.nextLine().trim().toLowerCase();
                continuer = rep.equals("o") || rep.equals("oui");
            }

        } catch (Exception e) { e.printStackTrace(); }
        finally { scanner.close(); }
    }
}
/*
 * client_voyelle.c — C CORBA client using TAO ORB
 * Compiled from the same VoyelleService.idl as the Java server.
 * Calls compterVoyelles() on the Java server via IIOP — no Java needed.
 */

#include "VoyelleServiceC.h"   /* generated by tao_idl from VoyelleService.idl */
#include "orbsvcs/CosNamingC.h"
#include 
#include 

int main(int argc, char* argv[]) {

    /* 1 — Initialize the ORB */
    CORBA_Environment env = CORBA_ENVIRONMENT_INIT;
    CORBA_ORB orb = CORBA_ORB_init(&argc, argv, "mico-local-orb", &env);

    /* 2 — Resolve the Naming Service */
    CORBA_Object nsObj = CORBA_ORB_resolve_initial_references(
        orb, "NameService", &env);
    CosNaming_NamingContext ns =
        CosNaming_NamingContext__narrow(nsObj, &env);

    /* 3 — Build the name path and resolve the service */
    CosNaming_NameComponent name[1];
    name[0].id   = "VoyelleService";
    name[0].kind = "";
    CosNaming_Name path = {1, 1, name, CORBA_FALSE};

    CORBA_Object svcObj =
        CosNaming_NamingContext_resolve(ns, &path, &env);

    VoyelleApp_VoyelleService svc =
        VoyelleApp_VoyelleService__narrow(svcObj, &env);

    printf("✓ CORBA service found (Java server)\n\n");

    /* 4 — Call the remote method */
    char input[256];
    int  continuer = 1;

    while (continuer) {
        printf("Enter a string: ");
        fgets(input, sizeof(input), stdin);
        input[strcspn(input, "\n")] = '\0'; /* strip newline */

        CORBA_long result =
            VoyelleApp_VoyelleService_compterVoyelles(svc, input, &env);

        printf("→ Vowels: %d\n\n", (int)result);

        printf("Continue? (y/n): ");
        char rep[4];
        fgets(rep, sizeof(rep), stdin);
        continuer = (rep[0] == 'y' || rep[0] == 'Y');
    }

    /* 5 — Release resources */
    CORBA_Object_release(svc, &env);
    CORBA_ORB_destroy(orb, &env);
    return 0;
}

What I Learned

This project pushed my understanding of distributed middleware to a new level. CORBA's IDL-first approach taught me how a language-neutral contract can power interoperability between completely different runtimes. Writing both a Java and a C client against the same Java server made the ORB and IIOP protocol concrete — not just theoretical. I also understood the role of the POA (Portable Object Adapter) in managing object lifecycles on the server side, which is more explicit and powerful than RMI's model.
