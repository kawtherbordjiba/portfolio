Project 04

Absence Management System

Java Swing GUI MySQL JDBC DAO Pattern MVC

Overview

A full desktop application for managing student attendance in an academic institution, built with Java Swing for the GUI and MySQL for persistent storage via JDBC. The system supports two user roles — Admin and Superviseur — and covers the complete lifecycle of absence tracking: recording, updating, justifying, and reporting. The architecture follows the DAO (Data Access Object) pattern to cleanly separate business logic from database operations.

Screenshots

Key Features

  • Role-based authentication — Admin and Superviseur access levels
  • Full CRUD for students, filieres, sections and supervisors
  • Absence recording with status: Présent, Absent, Retard, Excusé
  • Dashboard with real-time stats — daily & monthly attendance rates
  • Top absent students report with per-month aggregation
  • Search and filter across all entities
  • Duplicate absence detection before inserting a new record
  • DAO pattern — each entity has its own dedicated data access class

What I Learned

This project was my first complete desktop application combining a graphical interface with a real relational database. I learned how to structure a multi-layered Java application using the DAO pattern, how to manage JDBC connections properly with try-with-resources, and how to design a normalized schema with foreign key constraints. Building the dashboard stats with SQL aggregations (CASE WHEN, GROUP BY, date filtering) gave me strong practical SQL skills beyond basic CRUD.
