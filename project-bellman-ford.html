Project 09

Bellman-Ford Algorithm Program

C Graph Algorithms Shortest Path Negative Weights Cycle Detection

Overview

Implementation of the Bellman-Ford shortest path algorithm in C, using an edge list representation. Unlike Dijkstra's algorithm, Bellman-Ford can handle graphs with negative edge weights and is also capable of detecting negative weight cycles — making it essential for applications such as network routing protocols (RIP), arbitrage detection in finance, and constraint graphs. The program computes shortest distances from a source vertex and reports if a negative cycle exists.

Bellman-Ford vs Dijkstra

Dijkstra

  • Greedy — picks min distance vertex
  • O(V²) or O(E log V) with heap
  • No negative edge weights
  • No negative cycle detection
  • Faster in practice

Bellman-Ford

  • Relaxation — iterates all edges V-1 times
  • O(V × E) always
  • Handles negative edge weights
  • Detects negative cycles
  • More versatile, used in RIP protocol

How the Algorithm Works

1
Initialize distances
Set dist[source] = 0, all other distances = ∞. Store predecessor array for path reconstruction.
2
Relax all edges — V−1 times
For every edge (u, v, w): if dist[u] + w < dist[v], update dist[v]. After V−1 passes, all shortest paths are found.
3
Detect negative cycles
Run one more pass over all edges. If any distance still decreases, a negative cycle exists — there is no finite shortest path.
4
Output results
Print shortest distance to each vertex and reconstruct each path through the predecessor array.

Key Features

  • Edge list representation — simpler than adjacency matrix for sparse graphs
  • Handles negative edge weights — a key advantage over Dijkstra
  • Negative cycle detection via an extra relaxation pass after V−1 iterations
  • Predecessor array to reconstruct the full shortest path to any vertex
  • Prints "No path" for vertices unreachable from the source

Core Implementation

#include <stdio.h>
#include <limits.h>

#define MAX_V 100
#define MAX_E 500
#define INF   INT_MAX

typedef struct {
    int src, dest, weight;
} Edge;

Edge  edges[MAX_E];
int   dist[MAX_V], pred[MAX_V];
int   V, E;

void bellmanFord(int source) {
    /* Step 1 — Initialize */
    for (int i = 0; i < V; i++) { dist[i] = INF; pred[i] = -1; }
    dist[source] = 0;

    /* Step 2 — Relax all edges V-1 times */
    for (int i = 0; i < V - 1; i++) {
        for (int j = 0; j < E; j++) {
            int u = edges[j].src, v = edges[j].dest, w = edges[j].weight;
            if (dist[u] != INF && dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pred[v] = u;
            }
        }
    }

    /* Step 3 — Detect negative weight cycle */
    for (int j = 0; j < E; j++) {
        int u = edges[j].src, v = edges[j].dest, w = edges[j].weight;
        if (dist[u] != INF && dist[u] + w < dist[v]) {
            printf("⚠ Negative weight cycle detected — no solution.\n");
            return;
        }
    }

    /* Step 4 — Print results */
    printf("Vertex\tDistance\tPath\n");
    for (int i = 0; i < V; i++) {
        if (i == source) continue;
        if (dist[i] == INF) { printf("%d\tUnreachable\n", i); continue; }
        printf("%d\t%d\t\t", i, dist[i]);
        /* Reconstruct path */
        int path[MAX_V], len = 0, cur = i;
        while (cur != -1) { path[len++] = cur; cur = pred[cur]; }
        for (int k = len - 1; k >= 0; k--)
            printf("%d%s", path[k], k ? " → " : "\n");
    }
}

int main() {
    printf("Enter vertices and edges: "); scanf("%d %d", &V, &E);
    for (int i = 0; i < E; i++)
        scanf("%d %d %d", &edges[i].src, &edges[i].dest, &edges[i].weight);
    int src; printf("Source vertex: "); scanf("%d", &src);
    bellmanFord(src);
    return 0;
}

What I Learned

Implementing Bellman-Ford right after Dijkstra gave me a clear comparative understanding of when each algorithm is appropriate. The key insight was that Dijkstra's greedy approach breaks with negative weights, while Bellman-Ford's exhaustive edge relaxation guarantees correctness even then. The negative cycle detection step — running one extra relaxation pass and checking if any distance still improves — was a particularly elegant idea that deepened my appreciation of algorithm design.
